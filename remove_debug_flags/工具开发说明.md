# RTL Debug 标记清理工具开发说明

## 模块结构

- `remove_debug_flags.py`：清理核心、CLI 和 Tkinter GUI。
- `test_remove_debug_flags.py`：核心转换、VHDL 规则和备份目录结构的单元测试。
- `run_remove_debug_flags_gui.bat`：Windows GUI 启动脚本。

## 核心处理流程

1. 按扩展名递归收集 `.v`、`.sv`、`.vh`、`.svh`、`.vhd`、`.vhdl` 文件。
2. 使用 BOM/UTF-8/GB 系列候选编码读取源码；只有原文件确实带 BOM 时才按 BOM 编码写回，普通 UTF-8 文件保持无 BOM。
3. Verilog/SystemVerilog 通过轻量 lexer 区分代码区、行注释、块注释和字符串。
4. 仅在代码区识别 `(* ... *)` 属性块，并支持相邻属性块成组处理。
5. VHDL 采用行级 attribute 语句识别，删除 `mark_debug` 及同对象配套 `dont_touch`、`keep`。
6. 写回前按源目录相对路径复制备份到固定的并行 `debug_version`，不提供用户自定义备份根目录入口。

## dont_touch 策略

默认策略是仅移除伴随 `MARK_DEBUG` 出现的 `dont_touch` 与 `KEEP/keep`。独立 `dont_touch`、`KEEP/keep` 可能承载综合约束意图，因此默认保留。显式启用 `--remove-all-dont-touch` 或 GUI 对应选项后仅对 `dont_touch` 执行全量移除；`KEEP/keep` 仍保持“仅伴随删除”策略。

## 格式策略

默认只做局部空白整理，目的为消除属性删除后的多余空白。不执行全文件格式化，不重排声明块，不更改注释正文。
