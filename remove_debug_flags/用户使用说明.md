# RTL Debug 标记清理工具使用说明

## 目的

本工具用于递归扫描 RTL 源码目录，清除 Vivado 调试属性 `MARK_DEBUG`，并按配置清除与其伴随出现的 `dont_touch`、`KEEP/keep`。处理目标是源码级 debug 标记清理，不修改 RTL 逻辑、端口连接、表达式或时序行为。

## 启动方式

双击执行：

```bat
run_remove_debug_flags_gui.bat
```

命令行预览：

```powershell
python .\remove_debug_flags.py .\rtl --check
```

命令行写回并备份：

```powershell
python .\remove_debug_flags.py .\rtl --apply --backup
```

## GUI 操作流程

1. 选择源目录，例如 `remove_debug_flags\rtl`。
2. 查看备份目录提示。备份目录固定为源目录同级的 `debug_version`，不可在界面中修改。
3. 保持“写回修改”未勾选，先执行预览扫描。
4. 查看日志中的命中文件和统计结果。
5. 确认无误后勾选“写回修改”和“写回前备份到 debug_version”。
6. 再次点击“开始”，确认弹窗后执行写回。

## 备份规则

默认备份目录固定为源目录同级 `debug_version`，目录结构保持与源目录内部结构一致。

示例：

```text
源文件:
rtl\arinc818_src\dvi_final_out_0.v

备份文件:
debug_version\arinc818_src\dvi_final_out_0.v.bak
```

若备份文件已存在，会自动生成 `.bak1`、`.bak2` 等后缀，不覆盖已有备份。

## 清理规则

- Verilog/SystemVerilog：识别并清除 `(* MARK_DEBUG="true" *)`、`(* MARK_DEBUG = "TRUE" *)` 等属性写法。
- VHDL：识别并清除 `attribute mark_debug : string;` 和 `attribute mark_debug of ... is ...;`。
- 默认只清除与 `MARK_DEBUG` 同一属性块或相邻属性块绑定的 `dont_touch`、`KEEP/keep`。
- 独立 `dont_touch`、`KEEP/keep` 默认保留。
- 勾选“移除全部 dont_touch”后，所有识别到的 `dont_touch` 都会被清理。
- 注释和字符串中的 `MARK_DEBUG` 不会被处理。
- 普通 UTF-8 文件写回后仍保持无 BOM，避免 Vivado 在文件开头显示 BOM 乱码。

## 风险边界

工具只进行 debug 属性文本清理和局部空白整理，不做全文件格式化，不改 HDL 语义 token，不调整端口顺序，不修改表达式。写回前建议保持默认备份选项。
