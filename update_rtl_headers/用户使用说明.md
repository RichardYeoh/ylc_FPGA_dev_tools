# 用户使用说明

## 1. 工具用途

本工具用于批量处理 `rtl/` 目录中的 Verilog/SystemVerilog 源文件头注释，自动完成以下工作：

- 删除版权字段
- 删除作者邮箱字段
- 将符合规则的作者名统一改为 `ylcheng`
- 处理普通 `AUTHOR` 字段
- 处理 `Release history` 表格中的 `AUTHOR` 列

推荐直接双击的入口文件：

[run_update_rtl_headers.bat](/E:/proj/ag_other/sources_1/run_update_rtl_headers.bat)

核心脚本文件位置：

[update_rtl_headers.py](/E:/proj/ag_other/sources_1/update_rtl_headers.py)

## 2. 适用文件

脚本会递归处理目标目录下的以下文件：

- `.v`
- `.vh`
- `.sv`
- `.svh`

默认处理目录为当前工程下的 `rtl/`。

## 3. 使用前要知道的规则

### 3.1 会删除什么

以下字段会删除：

- `Copyright`
- `CopyRight`
- `AUTHOR'S EMAIL`
- `Email`

如果版权行后面紧跟着类似下面这种声明，也会一起删除：

- `Confidential Proprietary`
- `All Rights Reserved`

### 3.2 会修改什么

以下字段中的作者值，若命中匹配规则，会被统一改成 `ylcheng`：

- `AUTHOR`
- `Author`
- `Author/Designer`
- `Release history` 表中的 `AUTHOR` 列

### 3.3 什么情况下会改成 `ylcheng`

脚本不会无脑替换所有作者名。

它会先把作者内容：

1. 转成小写
2. 去掉非字母字符
3. 与 `yanglicheng` 和 `lichengyang` 做相似度匹配

如果匹配公共字母数大于 8，就替换为 `ylcheng`。

通常以下内容会被替换：

- `Yang Licheng`
- `YangLicheng`
- `lichengyang`
- `ylcheng@actri.avic`

通常以下内容不会被替换：

- 空字段
- 其他英文名
- 中文姓名

## 4. 最简单用法

直接双击：

[run_update_rtl_headers.bat](/E:/proj/ag_other/sources_1/run_update_rtl_headers.bat)

双击后会自动完成：

1. 对 `rtl/` 执行实际修改
2. 自动复测一次
3. 在窗口中打印处理信息
4. 停在结果界面，等待你按键关闭

这就是推荐的标准用法。

## 5. 使用前必须知道

`run_update_rtl_headers.bat` 不会弹出确认框。

也就是说：

- 双击后会直接修改 `rtl/` 中命中的文件
- 不会先询问你是否继续
- 如果你只想先看有哪些文件会被改，请不要双击 BAT，而应使用下面的命令行检查模式

## 6. 正确使用方法

### 方式一：双击直接执行

直接在资源管理器中双击：

[run_update_rtl_headers.bat](/E:/proj/ag_other/sources_1/run_update_rtl_headers.bat)

执行结束后，窗口中通常能看到以下几类信息：

- 处理根目录
- 扫描到的源文件数量
- 实际修改的文件数量
- 自动复测结果
- 命中的文件路径列表

如果最终复测结果显示：

```text
[CHECK] would update 0 file(s)
```

说明当前目录下已处理完成。

### 方式二：命令行手工执行

适合希望先检查、再决定是否写回的用户。

#### 第一步：进入工程目录

在 PowerShell 中进入工程根目录：

```powershell
cd E:\proj\ag_other\sources_1
```

#### 第二步：先做检查，不直接改文件

先运行检查模式：

```powershell
python .\update_rtl_headers.py .\rtl
```

这一步不会修改文件，只会告诉你哪些文件会被处理。

如果输出类似：

```text
would update 26 file(s)
```

表示有 26 个文件命中规则。

#### 第三步：确认后执行实际改写

执行：

```powershell
python .\update_rtl_headers.py --apply .\rtl
```

执行后会直接写回文件。

如果输出类似：

```text
updated 26 file(s)
```

表示实际修改了 26 个文件。

#### 第四步：再次复测

执行：

```powershell
python .\update_rtl_headers.py .\rtl
```

如果结果是：

```text
would update 0 file(s)
```

说明脚本已经把当前目录下所有可处理文件处理完毕，结果达到幂等。

## 9. 常见问题

### 为什么脚本说 `would update N file(s)`，但文件还没变

因为你运行的是检查模式，没有带 `--apply`。

### 为什么双击 BAT 后文件直接变了

因为 BAT 入口就是“直接执行实际改写”的模式，它不会再次确认。

### 为什么复测结果不是 0

可能原因：

- 你只运行了检查模式，没有真正写回
- `rtl/` 目录被恢复过
- 你在运行后又手工改回了文件
- 有新的文件被加入到目录中

### 为什么某个作者字段没有被替换

常见原因：

- 它不在文件头连续注释区内
- 它不是 `AUTHOR` / `Author` / `Author/Designer` / `Release history AUTHOR` 这几类目标位置
- 它与目标名字的相似度不够高

### 为什么某些带邮箱或附加字符的作者也会被替换

因为当前规则采用的是“公共字母数大于 8”的宽匹配，而不是严格全字匹配。只要字母重合度足够高，就会命中。

## 10. 安全建议

- 首次使用时，如果你不确定结果，先运行检查模式
- 如有需要，先自行备份 `rtl/`
- 批量处理后务必再次执行复测
- 对敏感工程建议配合版本管理一起使用
