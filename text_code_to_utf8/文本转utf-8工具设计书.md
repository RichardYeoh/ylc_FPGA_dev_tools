# 文本转 UTF-8 工具设计书

## 1. 目标
为工程目录提供一个简洁的可视化工具，递归扫描所选目录及子目录中的文本文件，若文件包含中文且编码不是 UTF-8，则直接转换为 UTF-8 并覆盖写回。提供进度显示、详细日志、开始前确认提示、完成提示对话框。

## 2. 适用范围
适用于可由文本编辑器打开和编辑的文件（源代码、脚本、配置、说明文档等）。工具默认以扩展名过滤文本文件，同时支持“无后缀文件（若为文本）”的可选处理。

## 3. 设计原则
- 安全直观：开始前二次确认，处理完成弹窗提示。
- 可追溯：日志记录每个文件的处理结论与统计汇总。
- 低依赖：仅使用 Python 标准库与 Tkinter，避免外部依赖。
- 稳健：跳过疑似二进制文件，减少误处理风险。

## 4. 功能拆分
1. UI 层
   - 目录选择
   - 开始转换按钮
   - 进度条与状态文本
   - 详细日志窗口
   - 开始前确认对话框
   - 完成提示对话框
2. 扫描与过滤
   - 递归遍历 `os.walk`
   - 仅处理文本扩展名或无后缀文本（可选）
   - 二进制检测（包含 `\x00` 或过多控制字符）
3. 编码识别与转换
   - 先尝试 UTF-8 解码
   - 若 UTF-8 失败，按候选编码列表解码（`gb18030`、`gbk`、`gb2312`、`big5`）
   - 仅当内容包含中文时才进行覆盖写入
4. 日志与统计
   - 日志记录每个文件的处理结果
   - 汇总统计：总候选、已处理、已转换、已是 UTF-8、无中文跳过、疑似二进制、解码失败、扩展名跳过

## 5. 编码与中文判定策略
- 中文判定范围：CJK 基本区 `U+4E00–U+9FFF`，扩展 A `U+3400–U+4DBF`。
- 编码判定：
  - UTF-8 可严格解码则视为 UTF-8。
  - UTF-8 解码失败时，依次尝试候选编码。
  - 候选编码解码成功且包含中文，才转换为 UTF-8。

## 6. UI 交互流程
1. 选择目录
2. 点击开始
3. 弹出确认对话框
4. 递归扫描并处理文件，日志实时输出，进度条更新
5. 处理完成弹窗提示

## 7. 关键实现文件
- `text_to_utf8_tool.py`

## 8. 风险与限制
- 仅基于启发式判断文件是否为文本，极少数二进制文件可能误判为文本（已通过二进制检测降低风险）。
- 编码识别采用候选列表，不保证覆盖所有编码；如出现解码失败会记录日志并跳过。
