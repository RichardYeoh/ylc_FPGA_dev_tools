# 文本编码转换工具设计书

## 1. 目标
为工程目录提供一个低依赖、可直接运行的编码整理工具。工具既支持目录递归扫描，也支持单文件定点转换；仅当文件内容包含中文且当前编码不是目标编码时才执行覆盖写回。当前目标编码支持 `UTF-8` 与 `GB2312` 两种单选模式，并在界面上实时反馈进度、统计与详细日志。

## 2. 本次改进重点
- 增加“目录递归 / 单文件”双模式入口。
- 增加“目标编码”单选项，可在 `UTF-8` 与 `GB2312` 间切换。
- UI 从基础表单改为更现代的卡片式布局，增加结果概览区。
- 增强编码识别，补上 UTF-16 / UTF-32 BOM 文件识别能力。
- 增加无 BOM UTF-16 启发式判断，降低被误判成二进制的概率。
- 引入文本质量评分，降低候选编码误解码后的误转风险。
- 增加“停止任务”和“转换前创建 `.bak` 备份”选项。

## 3. 功能范围
1. 处理目标
   - 目录模式：递归扫描所选目录及全部子目录。
   - 单文件模式：仅转换一个指定文件。
2. 文件筛选
   - 目录模式按常见文本扩展名过滤。
   - 可选处理无后缀文件。
   - 单文件模式不做扩展名限制，但仍做二进制与编码判定。
3. 编码转换
   - 识别 UTF-8、UTF-8 BOM、UTF-16、UTF-32。
   - 候选识别 `gb18030`、`gbk`、`gb2312`、`big5`。
   - 目标编码单选：`UTF-8` 或 `GB2312`。
   - 仅当解码后包含中文且不是目标编码时才写回目标编码。
   - 当目标为 `GB2312` 且文本存在无法编码字符时，记录失败并跳过写回。
4. 运行控制
   - 开始前确认对话框。
   - 后台线程执行，主线程负责 UI 刷新。
   - 支持处理中请求停止。
5. 输出反馈
   - 实时日志。
   - 进度条与状态文本。
   - 执行中统计卡片。
   - 完成/取消弹窗汇总。

## 4. 总体结构
### 4.1 UI 层
- 基于 Tkinter / ttk。
- 顶部使用强调色标题区突出工具定位。
- 主体分为“处理目标”“执行控制”“详细日志”“处理选项”“结果概览”“识别增强”六块卡片。

### 4.2 处理引擎
- `ConversionOptions`：封装运行模式、目标路径、目标编码、无后缀处理和备份选项。
- `ConversionStats`：统一累计总候选、已处理、已转换、已是目标编码、跳过、失败和备份数。
- `process_targets()`：统一调度目录模式和单文件模式。
- `process_one_file()`：完成单文件读取、解码判定、备份与写回。

### 4.3 并发模型
- 工作线程执行扫描与转换。
- UI 线程仅处理队列消息，避免后台线程直接操作 Tk 控件。
- 使用 `log_queue`、`progress_queue`、`finish_queue` 分离日志、进度和结束事件。

## 5. 编码识别策略
### 5.1 优先顺序
1. BOM 识别
2. UTF-8 严格解码
3. 无 BOM UTF-16 启发式判断
4. 中文场景常见编码候选列表
5. 目标编码一致性判定

### 5.2 UTF-16 / UTF-32 处理
- 对带 BOM 的 UTF-16 / UTF-32 文件优先用 BOM 驱动的标准 codec 解码，避免 BOM 残留影响文本质量判断。
- 对无 BOM UTF-16 文件，通过偶数字节 / 奇数字节中的 `0x00` 分布做启发式推断。

### 5.3 文本质量评分
- 统计可打印字符比例、控制字符比例、换行数量和中文数量。
- 仅当文本质量达到阈值时，候选解码结果才被接受。
- 该策略主要用于降低“字节流虽然能解码，但结果明显不像正常文本”的误判风险。

### 5.4 目标编码判定与写回
- 目标为 `UTF-8` 时，区分普通 UTF-8 与 `UTF-8 BOM`。
- 目标为 `GB2312` 时，通过“按 `GB2312` 解码后是否与已识别文本一致，且可按 `GB2312` 原样回编码”来判断是否已经是目标编码。
- 写回前统一用目标编码重新编码；若 `GB2312` 编码失败，则记录“目标编码不支持”并跳过。

## 6. 文件安全策略
- 默认仍保留“直接改写”的原始行为。
- 当用户勾选备份时，先复制出 `.bak` 文件，再写入目标编码。
- 如果备份失败，则跳过该文件，避免在安全措施失效时继续覆盖写回。

## 7. 日志与统计
- 日志逐文件输出以下状态：
  - `[已转换]`
  - `[已是目标编码]`
  - `[已是目标编码(BOM)]`
  - `[无中文-跳过]`
  - `[跳过-疑似二进制]`
  - `[读取失败]`
  - `[写入失败]`
  - `[解码失败]`
  - `[目标编码不支持-跳过]`
  - `[已备份]`
- 统计汇总包括：
  - 总候选文件
  - 已处理文件
  - 目标编码
  - 已转换
  - 已是目标编码
  - 已是目标编码(BOM)
  - 无中文跳过
  - 疑似二进制跳过
  - 扩展名跳过
  - 读取失败
  - 写入失败
  - 解码失败
  - 目标编码不支持
  - 已创建备份
