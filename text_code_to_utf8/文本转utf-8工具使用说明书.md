# 文本编码转换工具使用说明书

## 1. 工具简介
本工具用于把文本文件整理为指定编码。当前支持 `UTF-8` 与 `GB2312` 两种目标格式，支持目录递归扫描和单文件处理两种模式，适合工程代码、脚本、文档和配置文件的统一编码整理。

## 2. 运行环境
- Windows
- Python 3（推荐 3.8 及以上）
- 无需额外安装第三方依赖

## 3. 启动方式
在 `text_code_to_utf8` 目录中任选一种方式启动：

```powershell
python text_to_utf8_tool.py
```

或双击：

`一键启动.bat`

## 4. 界面说明
### 4.1 处理目标
- `目录递归`：扫描选定目录及其所有子目录。
- `单文件`：仅处理一个文件。

### 4.2 处理选项
- `目标编码`：单选 `UTF-8` 或 `GB2312`。
- `目录模式包含无后缀文件`：目录模式下允许处理没有扩展名的文本文件。
- `转换前创建 .bak 备份`：转换前先备份原文件，适合更谨慎的场景。

### 4.3 执行控制
- `开始转换`：开始执行任务。
- `停止任务`：处理中请求提前停止。

### 4.4 结果概览
界面右侧会持续刷新：
- 总候选
- 已处理
- 已转换
- 已是目标编码
- 失败数
- 跳过数和备份数

## 5. 使用步骤
### 5.1 目录模式
1. 选择 `目录递归`。
2. 点击 `选择目录`，选中待处理目录。
3. 按需勾选无后缀文件和备份选项。
4. 选择目标编码 `UTF-8` 或 `GB2312`。
5. 点击 `开始转换`。
6. 在确认框中确认执行。
7. 查看进度条、实时日志和统计卡片。

### 5.2 单文件模式
1. 选择 `单文件`。
2. 点击 `选择文件`，选中目标文件。
3. 选择目标编码。
4. 如需保留原文件，可勾选备份。
5. 点击 `开始转换` 并确认。

## 6. 日志状态说明
- `[已转换]`：已从原编码转成目标编码
- `[已是目标编码]`：文件已经是当前选定的目标编码
- `[已是目标编码(BOM)]`：文件以带 BOM 的目标编码形式存在，目前只在目标为 UTF-8 时出现
- `[无中文-跳过]`：文件不含中文，按规则跳过
- `[跳过-疑似二进制]`：疑似二进制文件，不参与转换
- `[读取失败]`：读取文件失败
- `[写入失败]`：写入目标编码失败
- `[解码失败]`：候选编码均未通过判定
- `[目标编码不支持-跳过]`：文本可解码，但无法编码成当前目标格式
- `[已备份]`：已生成 `.bak` 备份文件

## 7. 当前识别范围
- UTF-8
- UTF-8 BOM
- UTF-16
- UTF-32
- GB18030
- GBK
- GB2312
- Big5

## 8. 当前输出范围
- UTF-8
- GB2312

## 9. 注意事项
- 工具会直接覆盖文件；如果目标文件很重要，建议勾选 `.bak` 备份。
- 默认只处理“包含中文且不是目标编码”的文本文件。
- 单文件模式不做扩展名限制，但仍会跳过疑似二进制文件。
- 当目标编码为 `GB2312` 时，若文本中包含 `GB2312` 无法表示的字符，工具会记录失败并跳过该文件。
- 如果处理中点击 `停止任务`，程序会在当前文件处理结束后尽快退出本轮任务。
