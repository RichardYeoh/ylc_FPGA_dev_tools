# 仿真脚本生成工具开发说明书

## 1. 文档定位
本文面向维护者和二次开发者，说明当前版本工具的设计目标、UI 交互方式、路径处理逻辑、生成规则与维护边界。

## 2. 当前版本目标
当前版本的目标是：
- 提供可直接双击启动的简洁 UI
- 让用户手动选择要处理的 `questa/modelsim` 目录
- 生成 `my_sim.tcl` 与 `my_sim.bat`
- 将生成文件放在所选目录的上一级
- 保证从上一级目录启动时，Vivado 原始相对路径仍然正确
- 保留用户波形块和最后一条 `run ...`
- 当目标目录内已有同名 TCL 时，可回退继承其中的用户波形块

## 3. 非目标
当前版本不负责：
- 触发 Vivado 自动导出脚本
- 修改 RTL/TB 逻辑
- 提供回归调度、批量工程管理、测试报告聚合

## 4. 目录关系
工具目录建议放置在：

`<project>.srcs\sim_1\sim_do_auto_gen\`

用户在 UI 中选择的处理目录应为：

`<project>.sim\sim_1\behav\questa`

或：

`<project>.sim\sim_1\behav\modelsim`

生成结果写到：

`<project>.sim\sim_1\behav\`

## 5. 为什么输出改到上一级
Vivado 原始 `*_compile.do` 等文件内部大量使用相对路径，默认运行基准实际绑定在 `questa/modelsim` 目录。

如果只把旧内容简单搬到上一级目录直接运行，相对路径会失效。

因此当前版本采用如下策略：
1. 生成出的 `my_sim.tcl` 放在上一级目录
2. 启动 BAT 后先 `pushd` 到生成文件所在的上一级目录
3. TCL 以内建 `pwd` 作为基准，再拼出实际的 `questa/modelsim` 子目录
4. 显式 `cd` 到该子目录
5. 在该目录下执行整理后的 compile/elaborate/simulate 内容

这样既满足“文件放在上一级”，又保证 Vivado 的原始相对路径仍以 `questa/modelsim` 为基准。

## 6. 核心组成
- `gen_my_sim.ps1`
  - UI 入口
  - 目录校验
  - 源脚本解析
  - 旧配置继承
  - `my_sim.tcl` / `my_sim.bat` 生成
- `gen_my_sim.bat`
  - GUI 启动入口
  - 优先调用 `pwsh -STA`
  - 不可用时回退到 Windows PowerShell

## 7. UI 行为
UI 要点：
- 简洁布局，提供输入框、浏览按钮、生成按钮、关闭按钮、状态文字、日志框
- 手动浏览并选择目录
- 在日志框中打印处理进度
- 成功后弹出完成提醒
- 明确提示输出位于“上一级目录”
- 明确提醒不要把生成结果移回 `questa/modelsim`

## 8. 生成逻辑
### 8.1 `my_sim.tcl`
逻辑结构分为 7 部分：
- 启动保护段
  - `onerror {quit -code 1}`
  - 解析当前工作目录
  - 推导 `sim_dir`
  - 切换到 `sim_dir`
- Memory Init Staging 段
  - 来源：`*_compile.do` 解析到的 RTL 源文件
  - 处理：扫描固定字符串字面量形式的 `$readmemb/$readmemh`
  - 搜索：优先按 RTL 相对目录查找，再回溯到工程根目录做同名文件搜索
  - 歧义：若找到多个候选，UI 模式弹单选框，默认选中工具推荐项；无界面模式自动采用推荐项
- Compile 段
  - 来源：`*_compile.do`
  - 处理：去掉 `quit -force`
- Elaborate 段
  - 来源：`*_elaborate.do`
  - 处理：去掉空行、注释、`quit -force`
- Simulate Core 段
  - 来源：`*_simulate.do`
  - 处理：去掉 `wave/udo/view/run/quit`
- User Wave 段
  - 保留或继承用户标记区
- Runtime 段
  - 固定写入 `view wave` / `view structure` / `view signals`
  - 保留旧文件最后一条 `run ...`

### 8.2 `my_sim.bat`
逻辑：
- 进入脚本自身所在目录
- 打印当前目录和输出文件位置
- 从原始 `simulate.bat` 中解析 `bin_path`
- 对 `vsim` 路径做加引号处理，修复路径含空格问题
- 调用 `vsim -do "do {my_sim.tcl}" -l simulate.log`

## 9. 配置继承策略
### 9.1 用户波形块
优先级如下：
1. 上一级输出目录中已有的 `my_sim.tcl`
2. 目标 `questa/modelsim` 目录中已有的同名 `my_sim.tcl`
3. 默认模板块

### 9.2 运行时长
优先级如下：
1. 上一级输出目录中已有的 `my_sim.tcl` 的最后一条 `run ...`
2. 默认 `run 100ms`

## 10. 已修复问题
本版本已处理：
- 旧版 `my_sim.bat` 对 `%bin_path%/vsim` 未加引号，遇到空格路径会失败
- 旧版回读 UTF-8 无 BOM 输出文件时，Windows PowerShell 下可能破坏中文保留区
- 旧版默认要求在 `questa` 目录内运行，和实际使用预期不一致
- 旧版对 `do xxx_wave.do` / `do "xxx_wave.do"` / `do {xxx_wave.do}` 形式兼容性不足
- 新版可对部分 `$readmemb/$readmemh("裸文件名")` 场景自动搜索并同步初始化文件，降低 `vsim-7 Failed to open readmem file` 风险

## 11. 参数接口
当前支持：
- `-SimulatorDir`
- `-QuestaDir`
  - 作为兼容别名保留
- `-OutTclName`
- `-OutBatName`
- `-NoUi`

## 12. 编码策略
- 工具脚本使用 UTF-8
- 生成文件使用 UTF-8 无 BOM
- 文件回读使用 .NET `StreamReader(..., $true)`，优先正确处理 UTF-8 与 BOM

## 13. 维护建议
- 新增过滤规则时，优先保持默认行为兼容旧工程
- 若后续支持更多仿真器目录名，不要破坏“上一级输出 + 子目录切换”模型
- 每次修改后至少验证：
  - UI 能正常启动
  - 目录选择后能正确生成
  - 生成物位于上一级目录
  - 运行 BAT 时能进入正确子目录
  - 用户波形块和 run 行可跨次保留
  - 目标目录中旧版 TCL 的波形块回退继承可用
