# 仿真 DO 自动生成工具开发说明书

## 1. 文档定位
本说明书面向工具维护者和二次开发者，描述 `gen_my_sim.ps1` / `gen_my_sim.bat` 的设计目标、实现机制、扩展边界和维护规范。

适用场景：
- 需要在多工程间复用同一套 Questa 启动流程
- 需要在 Vivado 自动脚本基础上追加可维护的用户自定义波形区
- 需要稳定输出统一格式的 `my_sim.do` 与 `my_sim.bat`

## 2. 工具组成
目录建议：`<project>.srcs\sim_1\sim_do_auto_gen\`

核心文件：
- `gen_my_sim.ps1`：主生成器，负责解析来源脚本、拼接输出、保留用户区
- `gen_my_sim.bat`：Windows 启动入口，便于直接双击或命令行调用
- `仿真do脚本自动生成.md`：简版说明
- `工具开发说明书.md`：开发与维护说明（本文）
- `用户指导手册.md`：面向使用者的操作手册

## 3. 目标与非目标
目标：
- 自动读取 Vivado 导出的 `*_compile.do`、`*_elaborate.do`、`*_simulate.do`
- 合并为可直接运行且可重复更新的 `my_sim.do`
- 生成对应 `my_sim.bat`，统一调用 `vsim`
- 保留用户维护的波形块与 run 时长配置
- 避免工程名硬编码，支持跨工程直接复用

非目标：
- 不负责触发 Vivado 生成基础脚本
- 不修改 RTL / TB / 仿真行为逻辑
- 不提供完整回归平台功能（并发回归、报告聚合、CI 编排）

## 4. 前置条件
- 已安装可用的 Questa/ModelSim（命令 `vsim` 可调用）
- 已在 Vivado 中至少运行过一次 Behavioral Simulation
- 工程目录满足常规结构：`<project>.srcs` 与 `<project>.sim` 同级

## 5. 输入与输出定义
输入（来源文件）：
- `<project>.sim\sim_1\behav\questa\*_compile.do`
- `<project>.sim\sim_1\behav\questa\*_elaborate.do`
- `<project>.sim\sim_1\behav\questa\*_simulate.do`
- `<project>.sim\sim_1\behav\questa\simulate.bat`

输出（生成文件）：
- `<project>.sim\sim_1\behav\questa\my_sim.do`
- `<project>.sim\sim_1\behav\questa\my_sim.bat`

## 6. 路径推断策略（通用化关键）
当未传入 `-QuestaDir` 时，脚本按以下顺序自动识别：
1. 从脚本自身路径逆推：`sim_do_auto_gen -> sim_1 -> <project>.srcs`
2. 若命中 `<project>.srcs`，优先映射同级 `<project>.sim\sim_1\behav\questa`
3. 若同级有多个 `*.sim`，优先同名工程；否则取最近修改目录
4. 仍失败则报错，要求显式传入 `-QuestaDir`

这个策略使工具可复制到其他工程直接使用，无需修改工程名称。

## 7. 生成逻辑设计
`my_sim.do` 由 5 个逻辑段拼接：

A 段：Compile + 库映射
- 来源：`*_compile.do`
- 处理：基本原样保留，仅去除 `quit -force`

B 段：Elaborate
- 来源：`*_elaborate.do`
- 处理：去除空行、注释、`quit -force`

C 段：Simulate 启动核心
- 来源：`*_simulate.do`
- 处理：去除 `do *_wave.do`、`do *.udo`、`view`、`run`、`quit -force`
- 目的：把波形与时长控制移交给统一的 D/E 段

D 段：用户波形区
- 识别标记：
  - `#user wave-watch add here`
  - `#user wave-watch add here end`
- 若旧 `my_sim.do` 中存在标记区则原样保留，否则插入模板

E 段：视图与运行控制
- 固定写入：`view wave`、`view structure`、`view signals`
- `run ...` 行策略：优先沿用旧文件最后一条 `run`，无则默认 `run 100ms`

## 8. my_sim.bat 生成规则
- 优先从 `simulate.bat` 解析 `set bin_path=...`
- 若未找到，使用兜底默认值
- 统一输出启动命令：`vsim -do "do {my_sim.do}" -l simulate.log`
- 保留 `errorlevel` 分支，确保外部调用可判断成功/失败

## 9. 覆盖与幂等性
工具采用覆盖更新模式：
- 每次运行都会重写 `my_sim.do` 和 `my_sim.bat`
- 对同一输入重复执行，输出结构稳定（幂等）
- 通过用户标记块与 run 行复用机制保留人工维护内容

## 10. 参数接口
当前支持参数：
- `-QuestaDir`：显式指定目标 Questa 目录
- `-OutDoName`：输出 do 文件名，默认 `my_sim.do`
- `-OutBatName`：输出 bat 文件名，默认 `my_sim.bat`

扩展建议：
- 增加 `-RunTime` 显式覆盖 run 行
- 增加 `-WaveTemplate` 外部波形模板注入
- 增加 `-DryRun` 仅预览不落盘

## 11. 编码与兼容性
建议：
- `gen_my_sim.ps1` 使用 UTF-8 with BOM，提升 Windows PowerShell 中文兼容性
- 生成文件 `my_sim.do` / `my_sim.bat` 使用 UTF-8（脚本内显式写入）

注意：
- 若终端显示中文乱码，通常是控制台代码页问题，不代表文件内容损坏
- 可在终端执行 `chcp 65001` 后重试查看

## 12. 质量检查清单
每次修改后建议至少检查：
- 自动路径识别在两种工程目录下可用
- 显式 `-QuestaDir` 覆盖路径可用
- 能正确找到并拼接三类 do 文件
- 用户波形块可跨次运行保留
- run 行可跨次运行保留
- 输出 bat 可正确调用 vsim

## 13. 常见问题定位
问题：提示找不到 `*_compile.do`
- 原因：Vivado 尚未运行过仿真，或目录指向错误
- 处理：先在 Vivado 跑一次仿真，必要时显式传 `-QuestaDir`

问题：自动识别到错误工程
- 原因：同级存在多个 `*.sim`
- 处理：显式传 `-QuestaDir`，或规范目录命名

问题：`my_sim.bat` 启动失败
- 原因：`bin_path` 无效或 Questa 未安装
- 处理：检查 `simulate.bat` 的 `bin_path` 并重新生成

问题：中文注释显示异常
- 原因：终端编码页与文件编码不匹配
- 处理：切换 UTF-8 代码页或使用支持 UTF-8 的编辑器打开

## 14. 维护建议
- 将 `sim_do_auto_gen` 目录纳入版本管理
- 将用户波形块变更纳入 code review
- 跨工程复用时保留文件名一致，减少调用脚本维护成本
- 若引入新功能，优先保持默认行为不变，避免破坏现有流程
