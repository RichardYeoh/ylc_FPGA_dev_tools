# 仿真 DO 自动生成团队培训精简手册

## 1. 培训目标（10 分钟）
让团队成员掌握一套统一动作：
- 先由 Vivado 产出基础脚本
- 再用工具生成 `my_sim.do` / `my_sim.bat`
- 最后稳定复用、按需更新、快速排错

## 2. 一句话原则
先跑 Vivado 一次仿真，再运行 `gen_my_sim`，以后统一用 `my_sim.bat` 启动。

## 3. 标准目录
工具目录固定放在：
`<project>.srcs\sim_1\sim_do_auto_gen\`

## 4. 标准流程（SOP）
1. 在 Vivado 运行一次 Behavioral Simulation（必须）
2. 进入 `sim_do_auto_gen` 目录执行：
```bat
.\gen_my_sim.bat
```
3. 到 `<project>.sim\sim_1\behav\questa\` 执行：
```bat
.\my_sim.bat
```

## 5. 角色分工建议
- 设计工程师：维护 RTL/TB，触发 Vivado 脚本更新
- 仿真工程师：维护 `my_sim.do` 用户波形块与 run 时长
- 项目维护者：维护 `gen_my_sim.ps1` 版本和通用性

## 6. 可长期保留区（重点）
`my_sim.do` 中以下内容会在重生成时尽量保留：
- 用户波形标记区
- 最后一条 `run ...`

标记区模板：
```tcl
#user wave-watch add here
# add wave -position insertpoint sim:/<tb>/<path>/*
#user wave-watch add here end
```

## 7. 更新触发条件
出现以下任一情况，需重新执行 `gen_my_sim.bat`：
- RTL/TB 文件增删
- Vivado 重新导出仿真脚本
- 需要同步最新编译/展开/仿真入口命令

## 8. 快速排错（现场版）
问题：找不到 `*_compile.do`
- 结论：没先跑 Vivado 仿真
- 处理：先跑一次 Behavioral Simulation

问题：路径识别错工程
- 结论：同级有多个 `*.sim`
- 处理：改用 `-QuestaDir` 显式指定

问题：`my_sim.bat` 启动失败
- 结论：Questa 路径或 `bin_path` 异常
- 处理：检查 `simulate.bat` 后重新生成

## 9. 培训验收清单
1. 新人能独立执行一次完整流程（生成+运行）
2. 新人能在标记区添加 1 组自定义波形
3. 新人能解释“为什么必须先跑一次 Vivado 仿真”
4. 新人遇到 `*_compile.do` 报错能独立定位

## 10. 推荐口令（讲师开场）
“本工具不替代 Vivado，只负责把 Vivado 输出脚本转成可维护、可复用、可一键运行的团队标准入口。”
