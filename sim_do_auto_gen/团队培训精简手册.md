# 仿真脚本生成团队培训精简手册

## 1. 培训目标
让团队成员掌握新的统一动作：
- 先由 Vivado 生成原始脚本
- 再通过 UI 选择 `questa/modelsim` 目录
- 生成结果固定放在上一级目录
- 以后统一从上一级目录运行 `my_sim.bat`

## 2. 一句话原则
先跑 Vivado，一次生成；选中 `questa/modelsim`；以后在上一级 `behav` 目录启动。

## 3. 标准流程
1. 在 Vivado 中运行一次 Behavioral Simulation
2. 在 `sim_do_auto_gen` 目录执行：

```bat
.\gen_my_sim.bat
```

3. 在界面中选择：

`<project>.sim\sim_1\behav\questa`

或：

`<project>.sim\sim_1\behav\modelsim`

4. 到上一级目录执行：

```bat
cd <project>.sim\sim_1\behav
.\my_sim.bat
```

## 4. 培训重点
- 选的是 `questa/modelsim` 目录
- 生成物放的是上一级 `behav` 目录
- 运行也在 `behav` 目录
- 不要把生成出的 BAT 和 TCL 移回 `questa/modelsim`
- 若旧工程里已经有同名 `my_sim.tcl`，工具会尽量继承其中的用户波形块

## 5. 为什么必须这样做
因为 Vivado 原始脚本内部使用的是以 `questa/modelsim` 为基准的相对路径。

新工具通过包装 TCL 先切回子目录，再执行整理后的脚本内容，所以才能做到“文件放上一级、运行仍正确”。

## 6. 长期保留区
会尽量保留：
- 用户波形标记区
- 最后一条 `run ...`

标记区模板：

```tcl
#user wave-watch add here
# add wave -position insertpoint sim:/<tb>/<path>/*
#user wave-watch add here end
```

## 7. 快速排错
问题：选完目录后提示缺文件
- 结论：没先跑 Vivado 仿真，或者选错目录层级

问题：生成完双击不能跑
- 结论：检查上一级目录中的 `my_sim.bat` 是否仍放在 `behav` 这一层

问题：路径里有空格以前会挂
- 结论：当前版本已修复 BAT 路径加引号问题

## 8. 培训验收
1. 新人能独立完成一次目录选择和生成
2. 新人知道输出目录是上一级 `behav`
3. 新人知道为什么不能把生成结果移回 `questa/modelsim`
4. 新人能在用户波形区添加 1 组自定义波形
