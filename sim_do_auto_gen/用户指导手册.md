# 仿真 DO 自动生成工具用户指导手册

## 1. 手册目标
本手册面向日常仿真使用者，帮助你在任意 Vivado 工程中快速生成并运行 `my_sim.do` / `my_sim.bat`，减少手工拼接脚本和重复配置工作。

## 2. 你将获得什么
使用本工具后，你可以：
- 自动生成可复用的 `my_sim.do`
- 自动生成一键启动脚本 `my_sim.bat`
- 在反复更新脚本时保留自己的波形观察配置
- 在多工程中复用同一套方法，无需改工程名

## 3. 环境与前提
必须满足：
- 已安装 Vivado
- 已安装 Questa/ModelSim（可执行 `vsim`）
- 已在 Vivado 中至少运行过一次仿真（Behavioral Simulation）

为什么必须先在 Vivado 跑一次：
- 工具依赖 Vivado 先产出的 `*_compile.do`、`*_elaborate.do`、`*_simulate.do`
- 没有这些来源文件，就无法自动拼接

## 4. 目录放置要求
将以下文件放在：
`<project>.srcs\sim_1\sim_do_auto_gen\`

建议包含：
- `gen_my_sim.ps1`
- `gen_my_sim.bat`
- `仿真do脚本自动生成.md`
- `工具开发说明书.md`
- `用户指导手册.md`（本文）

## 5. 快速上手（推荐流程）
### 第一步：先在 Vivado 跑一次仿真
目标：让 Vivado 生成基础 do/bat 文件。

### 第二步：进入工具目录执行生成
在 `sim_do_auto_gen` 目录执行：

```bat
.\gen_my_sim.bat
```

或使用 PowerShell：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\gen_my_sim.ps1
```

### 第三步：运行生成的仿真脚本
进入输出目录（通常是 `<project>.sim\sim_1\behav\questa\`）执行：

```bat
.\my_sim.bat
```

## 6. 自动路径识别说明
默认不需要写工程名。脚本会自动推断：
- 从 `<project>.srcs` 推断到同级 `<project>.sim\sim_1\behav\questa`
- 若同级有多个 `*.sim`，优先同名工程，若无法区分则选择最近更新项

如果你想手动指定路径：

```powershell
.\gen_my_sim.ps1 -QuestaDir "D:\path\to\any_project.sim\sim_1\behav\questa"
```

## 7. 生成结果与更新机制
默认输出：
- `my_sim.do`
- `my_sim.bat`

输出位置：
- `<project>.sim\sim_1\behav\questa\`

更新机制：
- 每次运行都会覆盖旧的 `my_sim.do` / `my_sim.bat`
- 用于同步最新 Vivado 脚本内容
- 支持重复执行，无需手工清理

## 8. 你可以长期保留的自定义内容
在 `my_sim.do` 中，以下内容会尽量保留：
- 用户波形块（标记区）
- 最后一条 `run ...` 时长配置

标记区格式：
```tcl
#user wave-watch add here
# add wave -position insertpoint sim:/<tb>/<path>/*
#user wave-watch add here end
```

建议：
- 只在标记区内维护自定义波形
- 不要随意修改标记行文本本身

## 9. 常用操作场景
场景 A：RTL 文件增删后
1. 在 Vivado 重新运行一次仿真
2. 再执行 `gen_my_sim.bat` 重新生成
3. 运行 `my_sim.bat` 验证

场景 B：只想改仿真时长
1. 打开 `my_sim.do`
2. 修改最后一条 `run ...`
3. 后续重新生成时该 run 行会被尽量保留

场景 C：项目迁移到新工程
1. 复制 `sim_do_auto_gen` 文件夹到新工程对应位置
2. 在新工程 Vivado 先跑一次仿真
3. 执行工具，不需要改脚本里的工程名

## 10. 故障排查
问题：提示找不到 `*_compile.do`
- 处理：先在 Vivado 跑一次 Behavioral Simulation

问题：提示 `Questa directory does not exist`
- 处理：检查目录结构，或用 `-QuestaDir` 手动指定

问题：双击 `my_sim.bat` 无法启动
- 处理：检查 Questa 安装路径和 `simulate.bat` 中 `bin_path`

问题：文档或注释中文显示乱码
- 处理：切换终端到 UTF-8（`chcp 65001`）或用 IDE 直接查看

## 11. 最佳实践
- 每次 Vivado 更新仿真脚本后，都重新生成一次 `my_sim.do`
- 把自定义波形维护在标记区内，避免被覆盖
- 将工具目录纳入版本管理，便于团队共享
- 若工程较多，统一使用同样目录层级，减少手动指定路径

## 12. 命令速查
生成：
```bat
.\gen_my_sim.bat
```

手动指定 Questa 目录生成：
```powershell
.\gen_my_sim.ps1 -QuestaDir "D:\path\to\project.sim\sim_1\behav\questa"
```

运行仿真：
```bat
.\my_sim.bat
```
