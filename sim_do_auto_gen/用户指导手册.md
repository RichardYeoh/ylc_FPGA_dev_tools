# 仿真脚本生成工具用户指导手册

## 1. 快速上手
只看这 5 步就能用：

1. 先在 Vivado 里跑一次 Behavioral Simulation
2. 进入 `sim_do_auto_gen` 目录执行：

```bat
.\gen_my_sim.bat
```

3. 在界面里选择：

`<project>.sim\sim_1\behav\questa`

或：

`<project>.sim\sim_1\behav\modelsim`

4. 等待生成完成提示
5. 到上一级目录运行：

```bat
cd <project>.sim\sim_1\behav
.\my_sim.bat
```

## 2. 最重要的规则
- 选的是 `behav\questa` 或 `behav\modelsim`
- 生成出来的是上一级 `behav` 目录中的 `my_sim.tcl` 和 `my_sim.bat`
- 后续运行也在这个上一级 `behav` 目录
- 不要把生成出的 BAT 和 TCL 移回 `questa/modelsim`

## 3. 这套工具能做什么
- 通过 UI 手动选择 `questa/modelsim` 目录
- 自动生成 `my_sim.tcl` 与 `my_sim.bat`
- 生成文件固定放在所选目录的上一级
- 从上一级目录直接启动也能保证路径正确
- 尽量保留已有用户波形块和运行时长配置
- 会自动扫描部分 `$readmemb/$readmemh` 初始化文件引用，并尽量把所需文件同步到仿真运行目录
- 当同名初始化文件在工程里找到多个候选时，会弹出单选框让用户确认，默认选中工具推荐项

## 4. 必要前提
必须满足：
- 已安装 Vivado
- 已安装 Questa 或 ModelSim
- 已在 Vivado 中至少运行过一次 Behavioral Simulation

没有先跑过仿真时，本工具无法找到 `*_compile.do` 等来源文件。

## 5. 工具目录
请把工具目录放在：

`<project>.srcs\sim_1\sim_do_auto_gen\`

建议包含：
- `gen_my_sim.ps1`
- `gen_my_sim.bat`
- `仿真do脚本自动生成.md`
- `工具开发说明书.md`
- `用户指导手册.md`

## 6. 界面使用说明
界面包含：
- 目标目录输入框
- `浏览...` 按钮
- 一行中文提示
- `生成脚本` 按钮
- `关闭` 按钮
- 状态文字
- 运行日志窗口

生成过程中会打印处理进度；完成后会弹出提醒，并再次强调输出文件放置位置。

## 7. 输出与运行机制
默认输出到所选目录的上一级：

`<project>.sim\sim_1\behav\`

输出文件为：
- `my_sim.tcl`
- `my_sim.bat`

为什么这样设计：
- Vivado 原始脚本中的相对路径本质上仍以 `questa/modelsim` 为基准
- 新版 `my_sim.tcl` 会在运行时自动切回这个子目录
- 所以用户可以在上一级统一启动，而内部路径仍保持正确

## 8. 自定义内容的保留与继承
重新生成时，会尽量保留：
- 用户波形块
- 最后一条 `run ...`

波形块标记格式：

```tcl
#user wave-watch add here
# add wave -position insertpoint sim:/<tb>/<path>/*
#user wave-watch add here end
```

优先级说明：
- 优先继承上一级已有 `my_sim.tcl` 中的用户波形块
- 若上一级还没有 `my_sim.tcl`，则尝试继承目标 `questa/modelsim` 目录里同名 `my_sim.tcl` 中的用户波形块
- `run ...` 仍优先取上一级已有 `my_sim.tcl`

建议只在该标记区内维护自定义波形。

## 9. 常见场景
### 场景 A：RTL/TB 改动后
1. 在 Vivado 中重新跑一次仿真
2. 重新执行 `gen_my_sim.bat`
3. 到上一级输出目录运行 `my_sim.bat`

### 场景 B：只想修改运行时长
1. 打开上一级目录中的 `my_sim.tcl`
2. 修改最后一条 `run ...`
3. 下次重新生成时会尽量保留

### 场景 C：旧工程里已经有一份 `my_sim.tcl`
1. 先正常选择 `questa/modelsim` 目录
2. 生成器会优先使用上一级已有 `my_sim.tcl`
3. 如果上一级没有，则尝试继承目标目录中旧版同名 `my_sim.tcl` 的用户波形块

### 场景 D：迁移到新工程
1. 复制 `sim_do_auto_gen` 到新工程对应位置
2. 在新工程里先跑一次 Vivado 仿真
3. 执行 `gen_my_sim.bat`
4. 在 UI 中重新选择新工程的 `questa/modelsim` 目录

## 10. 故障排查
问题：提示找不到 `*_compile.do`
- 处理：先在 Vivado 跑一次 Behavioral Simulation

问题：选中的目录无效
- 处理：确认选中的是 `behav\questa` 或 `behav\modelsim`，而不是更上层目录

问题：`my_sim.bat` 启动失败
- 处理：检查 `simulate.bat` 中的 `bin_path`，并确认 Questa/ModelSim 已正确安装

问题：自定义中文注释在重生成后异常
- 处理：当前版本已修复 UTF-8 无 BOM 回读问题；若仍异常，请保留样例文件再排查

问题：工具提示检测到多个同名初始化文件候选
- 处理：按弹窗选择正确文件；若不确定，优先看工具默认推荐项，它通常会优先选择与当前 RTL 路径最近的候选

```

```

```bat

```
