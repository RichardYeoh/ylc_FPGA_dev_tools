# Codex Quota Taskbar 用户说明书

## 1. 工具用途

Codex Quota Taskbar 是一个 Windows 本地小工具，用于在任务栏附近实时显示 Codex/GPT Codex 配额状态：

- `5h`：5 小时窗口剩余额度。
- `W`：weekly 周窗口剩余额度。
- 每行右侧显示对应复位时间。

工具默认贴靠在 Windows 任务栏系统托盘左侧，只显示两行紧凑文字。默认开启鼠标穿透，不会影响任务栏按钮、托盘、网络、音量或时钟的点击。

## 2. 运行环境

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Windows |
| Python | 已安装 Python 3，且命令行可执行 `python` |
| PowerShell | Windows PowerShell 或 PowerShell 7 |
| Codex 登录状态 | 当前用户已有 Codex OAuth 状态，通常位于 `%USERPROFILE%\.codex\auth.json` |
| 项目路径 | `D:\proj\ylc_FPGA_dev_tools\codex_quota_taskbar` |

## 3. 快速启动

打开 PowerShell，进入项目目录：

```powershell
cd D:\proj\ylc_FPGA_dev_tools\codex_quota_taskbar
```

启动显示窗：

```powershell
.\scripts\run_app.ps1
```

启动 watcher：

```powershell
.\scripts\run_watcher.ps1
```

通常建议启动 watcher。watcher 会在检测到 Codex CLI、VS Code Codex/OpenAI/ChatGPT 扩展宿主或相关桌面进程后自动启动显示窗。

## 4. 开机自动运行

安装当前用户登录自启动：

```powershell
.\scripts\install_autostart.ps1
```

安装脚本会优先尝试注册 Windows 登录计划任务。如果权限不足，会自动回退到当前用户 Startup 文件夹快捷方式。

回退快捷方式位置通常为：

```text
C:\Users\liche\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\CodexQuotaTaskbarWatcher.lnk
```

取消开机自启动：

```powershell
.\scripts\uninstall_autostart.ps1
```

## 5. 关闭工具

只关闭任务栏显示窗，不关闭 watcher：

```powershell
.\scripts\stop_app.ps1
```

如果 watcher 仍在运行，并且 Codex 相关进程仍被检测到，显示窗可能会被 watcher 再次启动。

若要停止 watcher，可在任务管理器中结束对应的 `pythonw.exe`，命令行包含：

```text
codex_quota_watcher.py
```

## 6. 显示内容说明

默认显示两行：

```text
5h 94% 20:30
W  83% 06/16 11
```

含义如下：

| 显示项 | 说明 |
| --- | --- |
| `5h` | 5 小时配额窗口 |
| `W` | weekly 周配额窗口 |
| `94%` / `83%` | 剩余额度百分比 |
| `20:30` | 如果复位时间是当天，显示小时和分钟 |
| `06/16 11` | 如果复位时间不是当天，显示月/日和小时 |
| `--%` | 暂无有效余量数据 |
| `--:--` | 暂无有效复位时间 |
| `ERR` | 本轮读取或渲染出现错误 |

复位时间统一按香港时间 `UTC+08:00` 显示。

## 7. 刷新频率

当前默认配置：

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| `quota.refresh_seconds` | `2` | 显示窗每 2 秒刷新一次本地显示 |
| `quota.api_refresh_seconds` | `150` | 最多每 150 秒查询一次实时配额接口 |
| `watcher.poll_seconds` | `15` | watcher 每 15 秒扫描一次进程 |

这意味着界面会频繁刷新显示，但不会每 2 秒访问网络接口；实时接口结果会缓存 150 秒，以降低后台请求频率。

## 8. 修改配置

配置文件为：

```text
D:\proj\ylc_FPGA_dev_tools\codex_quota_taskbar\config.json
```

修改配置后，重新启动显示窗或 watcher 才能稳定生效。

### 8.1 修改接口查询间隔

将实时配额查询间隔改为 180 秒：

```json
"api_refresh_seconds": 180
```

建议不要设置得过低。源码中会把该值限制为至少 30 秒。

### 8.2 修改 watcher 扫描间隔

将进程扫描间隔改为 30 秒：

```json
"poll_seconds": 30
```

### 8.3 显示窗位置微调

如果文字距离系统托盘太近或太远，可以调整：

```json
"dock_offset_x": 4
```

数值增大时显示窗向右移动，数值减小时向左移动。

### 8.4 临时关闭鼠标穿透

默认鼠标穿透开启：

```json
"click_through": true
```

如果需要用右键关闭窗口或双击切换位置，可改为：

```json
"click_through": false
```

重新启动后：

- 右键显示窗：退出显示窗。
- 双击显示窗：在任务栏停靠和任务栏上方两种位置间切换。

## 9. 数据来源与回退

工具按以下优先级读取配额：

1. `provider_command` 外部命令输出。
2. Codex usage API。
3. 项目内 `quota_cache.json` 旧缓存。
4. 项目内 `quota_state.json` 手工状态文件。

如果实时 API 失败，但旧缓存仍存在，界面会继续显示旧缓存数据。

如果需要手工写入临时数据，可编辑 `quota_state.json`，格式示例：

```json
{
  "updated_at": "2026-06-11T20:00:00+08:00",
  "source": "manual",
  "windows": {
    "5h": {
      "remaining_fraction": 0.72,
      "remaining_text": "72%",
      "reset_at": "2026-06-11T22:00:00+08:00"
    },
    "weekly": {
      "remaining_fraction": 0.63,
      "remaining_text": "63%",
      "reset_at": "2026-06-16T11:00:00+08:00"
    }
  }
}
```

## 10. 常见问题

### 10.1 没有看到显示窗

先手动重启显示窗：

```powershell
.\scripts\run_app.ps1
```

该脚本会先停止本项目旧显示窗，再启动新显示窗，可恢复被隐藏的旧窗口。

### 10.2 Codex 关闭后显示窗还在

确认 watcher 配置：

```json
"close_when_codex_exits": true
```

如果 watcher 没有运行，则显示窗不会自动跟随 Codex 退出。

### 10.3 显示 `ERR`

可能原因：

- `config.json` JSON 格式错误。
- Codex 登录状态不可读。
- usage API 临时失败且缓存/手工状态也不可用。
- Python 环境异常。

可运行基础检查：

```powershell
python -m json.tool .\config.json
python -m py_compile .\src\codex_quota_provider.py .\src\codex_quota_taskbar.py .\src\codex_quota_watcher.py .\src\smoke_test.py
python .\src\smoke_test.py
```

### 10.4 计划任务安装失败

如果 `install_autostart.ps1` 显示计划任务注册失败，但随后提示已安装 Startup 快捷方式，这是正常回退路径。当前用户登录后仍会自动启动 watcher。

### 10.5 文字位置不合适

调整 `config.json` 中的：

```json
"dock_offset_x": 4
```

调整后重新启动：

```powershell
.\scripts\run_app.ps1
```

## 12. 卸载或清理

取消自启动：

```powershell
.\scripts\uninstall_autostart.ps1
```

关闭当前显示窗：

```powershell
.\scripts\stop_app.ps1
```

如需完全停止 watcher，在任务管理器中结束命令行包含 `codex_quota_watcher.py` 的 `pythonw.exe`。

## 13. 注意事项

- 工具会读取本机 `%USERPROFILE%\.codex\auth.json` 中的 Codex OAuth 状态，但不会把 token 写入项目缓存。
- `quota_cache.json` 只缓存标准化后的配额余量和复位时间。
- 不建议把 `provider_command` 指向不可信脚本。
- 修改配置后建议通过 `run_app.ps1` 或重新启动 watcher 让新配置生效。
