## 2026-06-04 初始化

- 用户要求：在 `E:\proj\ylc_FPGA_dev_tools` 下新建项目目录，开发一个常驻 Windows 底部任务栏的 Codex 配额提示工具，紧凑显示 `5h` 余量、周余量及复位时间，约占两个普通任务栏图标面积，并在打开 Codex CLI、VS Code 插件、桌面 app 时自动运行。
- 执行约束：在新建项目目录内开发并记录会话；遇到不明确或无法实现事项需先说明；按计划推进。
- 调研结论：官方帮助文档说明 Codex 限额需查看 Codex usage page 或 limit banner，本机 Codex CLI 0.137.0 公开命令未提供 `5h`/周余量输出；本机 `.codex` sqlite 状态库包含本地线程 token 累计，但不等价于官方 agentic quota。
- 初始实现决策：采用 Python 标准库实现，避免依赖未安装的 `.NET` 或第三方 GUI 包；默认实现为贴靠/覆盖任务栏区域的无边框置顶紧凑窗，而不是 Windows COM DeskBand；配额数据先通过 `quota_state.json` 或外部 `provider_command` 适配层提供。
- 追加调研：按用户提醒补查网络现成方案，发现 `codex-cli-usage`、LimitAID、CodexControl、Codex Quota Monitor 等工具或页面；其中 `codex-cli-usage` 明确说明调用 ChatGPT backend 的 Codex usage API，GitHub `openai/codex-plugin-cc#102` 也记录了 `primary_window`/`secondary_window` 字段。
- 调整决策：将 provider 改为优先调用 Codex usage API，使用项目内 `quota_cache.json` 做 5 分钟缓存，失败时回退到 `quota_state.json`；UI 和 watcher 逻辑保持不依赖第三方包。
- 实现结果：新增紧凑无边框置顶窗口、Codex 进程 watcher、PowerShell 运行脚本、自启动安装/卸载脚本、配置文件、配额 provider、README 和项目局部 `.gitignore`。
- 验证结果：`python -m py_compile` 通过；`python src\smoke_test.py` 通过，并从 `chatgpt-codex-usage-api` 读到真实配额快照。验证时缓存值为 `5h 73%`、`weekly 86%`，复位时间分别为 `2026-06-04T13:08:51+00:00` 和 `2026-06-11T03:08:39+00:00`。
- 运行状态：已通过 `scripts\run_watcher.ps1` 启动 watcher；当前检测到 `codex_quota_watcher.py` 和 `codex_quota_taskbar.py` 各一个 `pythonw.exe` 进程。
- 未执行项：未注册 Windows 登录自启动计划任务，因为该操作属于项目目录外的系统配置；已提供 `scripts\install_autostart.ps1` 和 `scripts\uninstall_autostart.ps1`。

## 2026-06-04 任务栏停靠修正

- 用户反馈：当前界面未显示；上次可见时覆盖了任务栏右侧状态显示区域，点击状态区域后界面消失；期望改为停靠在任务栏。
- 问题判断：旧实现是无边框置顶窗口按工作区和屏幕高度计算位置，实际属于覆盖任务栏区域的浮窗，不是任务栏子窗口，因此可能遮挡系统托盘、时钟、网络/音量等状态区域，也可能在任务栏交互后出现层级不稳定。
- 修改决策：将默认 `window.position` 从 `taskbar_overlay` 改为 `taskbar_docked`；主窗口启动后通过 `Shell_TrayWnd` 停靠到 Windows 任务栏窗口，递归查找 `TrayNotifyWnd`，并将提示窗放在系统托盘左侧 `dock_gap` 像素处；停靠失败时回退到任务栏上方紧凑浮窗。
- 兼容说明：双击窗口改为在“任务栏停靠”和“任务栏上方”之间切换；配额 API、缓存、watcher 逻辑未改动。
- 现场验证修正：Windows 11 + Tkinter 下 `SetParent` 对 `TkTopLevel` 不稳定，外层窗口会残留为普通顶层窗口；后续实现改为按 `Shell_TrayWnd` / `TrayNotifyWnd` 的真实屏幕坐标贴靠，不再依赖改父窗口。
- 用户截图反馈：位置基本正确，但黑色背景矩形和偏小字体不符合预期，且窗口点击区域会影响任务栏功能。
- 修改决策：默认启用透明背景、鼠标穿透、较大字体，并保留托盘左侧贴靠定位；关闭交互式右键/双击绑定，避免提示窗拦截任务栏点击。
- 配套调整：新增 `scripts\stop_app.ps1` 用于在鼠标穿透模式下停止显示窗；`.gitignore` 增加 `taskbar_capture*.png` 以排除本轮验证截图。
- 追加调整：用户反馈透明后字体看不清且希望位置更靠右；将常态字体颜色改为深色 `#111827`，低余量颜色改为深橙/深红，并新增 `dock_offset_x=18` 将显示内容向右偏移。
- 配色优化：根据 WCAG 普通文本对比度最低 4.5:1 和 Microsoft Fluent 的中性色主体、强调色少量使用原则，将单色文本改为三段式：`label_color=#0f6cbd`、`text_color=#1f2937`、`time_color=#4b5563`，低余量继续使用深橙/深红；位置进一步右移到 `dock_offset_x=32`。
- 追加调整：用户反馈三段式仍不美观；改回单色显示，常态色采用柔和深青蓝 `#0f3d4c`，保留低余量深橙/深红告警色。
- 2026-06-05 追加修正：用户反馈显示窗曾在某次任务栏/状态区域操作后消失，但进程仍可能存在。判断为顶层透明窗口被 Windows 桌面/任务栏交互隐藏的问题；在每次刷新定位后增加 `ShowWindow(SW_SHOWNOACTIVATE)` 和 `SetWindowPos(HWND_TOPMOST | SWP_SHOWWINDOW | SWP_NOACTIVATE)` 自恢复；`scripts\run_app.ps1` 改为先停止本项目旧 UI 再启动，避免“单实例还在但窗口不可见”。
- 2026-06-05 位置和配色微调：用户根据截图要求单色更协调、位置靠近右侧隐藏图标箭头；将常态色从深青蓝改为更中性的深灰蓝 `#334155`，窗口宽度从 `168` 缩到 `146`，列间距从 `2` 缩到 `1`，`dock_offset_x` 从 `32` 增到 `64`。
- 2026-06-05 抗锯齿和彩边修正：用户指出透明文字边缘像混入黑色/双层叠色，判断为 Tk `transparentcolor` 抠色导致抗锯齿边缘混入透明键色。参考 Fluent 中性色/材料底色和 Morandi 低饱和灰调思路，关闭透明抠色，改为接近浅色任务栏的背景 `#dceff5`，常态文字改为低饱和灰蓝 `#4f5f66`，告警色也改为灰调橙/红；位置从 `dock_offset_x=64` 回退到 `52`。
