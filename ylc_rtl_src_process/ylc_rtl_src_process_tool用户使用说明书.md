# ylc_rtl_src_process_tool 用户使用说明书

## 1. 工具用途

本工具用于批量处理 RTL / 文本文件：

- 文本编码转换（UTF-8 / GB2312）
- 可选执行头注释规范化（删除版权/邮箱、作者统一为 `ylcheng`）

适用于工程目录的例行整理与编码统一。

## 2. 运行环境

- Windows
- Python 3.8+（推荐 3.8 或以上）
- 无需额外依赖（仅使用标准库 + Tkinter）

## 3. 启动方式

在 `ylc_rtl_src_process` 目录中执行：

```powershell
python rtl_src_process_tool.py
```

或双击启动：

`一键启动.bat`

## 4. 界面说明

- 目标目录：默认指向工具目录下的 `rtl`
- 输出编码：`UTF-8` / `GB2312`
- 包含无后缀文件：若无后缀但为文本，可处理
- 执行头注释规则：按 update_rtl_headers 规则清理/统一头注释

## 5. 使用步骤

1. 点击“选择目录”，选择需要处理的工程目录
2. 选择目标编码（UTF-8 或 GB2312）
3. 如需处理无后缀文件，勾选“包含无后缀文件”
4. 如需执行头注释规则，勾选对应选项
5. 点击“开始处理”
6. 弹出确认警告后选择“确定”继续
7. 处理完成后查看日志和统计

## 6. 日志说明

日志中会出现以下标签：

- `[已转换]`：编码已转换为目标编码
- `[已转换+头注释更新]`：编码转换 + 头注释更新
- `[头注释更新]`：仅头注释更新
- `[已是UTF-8]` / `[已是GB2312]`：已是目标编码
- `[无中文]`：无中文内容，编码转换跳过
- `[疑似二进制]`：二进制文件跳过
- `[解码失败]`：候选编码均无法正确解码
- `[写入失败]`：目标编码无法表示或写入失败

## 7. 注意事项

- 工具会直接覆盖原文件，请先自行备份重要工程
- `GB2312` 目标编码可能无法表示全部字符，出现“写入失败”时该文件不会被修改
- 头注释处理只作用于文件开头连续注释区，不影响正文内容
- 对于非文本文件，会通过二进制检测自动跳过
