# ylc_rtl_src_process_tool 设计说明书

## 1. 目标与定位

`ylc_rtl_src_process_tool` 是一个独立运行的本地文件处理工具，用于批量处理工程目录下的 RTL 与相关文本文件。工具集成以下能力：

- 文本编码转换（UTF-8 / GB2312）
- 头注释规范化（可选，规则与 update_rtl_headers 保持一致）
- 图形界面（Tkinter），包含目录选择、确认弹窗、进度与日志

该工具不依赖于其他项目目录或脚本，放在任意位置均可直接运行。

## 2. 功能范围

### 2.1 编码转换

- 目标编码：`UTF-8` 或 `GB2312`（单选）
- 仅在检测到中文时才进行编码转换
- 对于已是目标编码的文件记录为“已是目标编码”并跳过

### 2.2 头注释处理（可选）

启用后，按以下规则处理文件头连续注释区：

- 删除 `Copyright` / `CopyRight`
- 删除紧随版权行后的保留声明：`Confidential`、`Proprietary`、`All Rights Reserved`
- 删除邮箱字段：`AUTHOR'S EMAIL` / `Email`
- 替换作者字段为 `ylcheng`
  - 目标字段：`AUTHOR` / `Author` / `Author/Designer`
  - `Release history` 表格中的 `AUTHOR` 列
  - 使用字母多重集交集计数，相似度阈值 > 8 视为命中

### 2.3 文件筛选

- 编码转换：按扩展名过滤（含常见源码/脚本/配置/文档），可选是否包含无后缀文件
- 头注释处理：启用时对全部文件生效，但会跳过疑似二进制文件

## 3. 结构与流程

### 3.1 主要模块

- UI 层（Tkinter）
- 扫描与过滤
- 二进制检测
- 编码探测与转换
- 头注释规范化
- 日志与统计

### 3.2 处理流程

1. 选择目标目录
2. 选择目标编码与是否启用头注释规则
3. 弹出确认警告
4. 递归扫描文件
5. 对每个文件执行：
   - 二进制检测
   - 解码（UTF-8 优先，失败后按候选编码列表尝试）
   - 头注释处理（可选）
   - 编码转换（仅在检测到中文时）
6. 日志与统计输出

## 4. 编码与中文判定策略

- 解码顺序：`utf-8` -> `gb18030` -> `gbk` -> `gb2312` -> `big5`
- 中文判定：CJK 基本区 `U+4E00–U+9FFF` + 扩展 A `U+3400–U+4DBF`
- 目标编码为 `GB2312` 时，若文本中包含该编码无法表示的字符，会记录为“写入失败”并跳过

## 5. UI 交互设计

- 目录选择
- 目标编码单选
- “包含无后缀文件”勾选
- “执行头注释规则”勾选
- 启动确认弹窗
- 进度条 + 状态文本
- 日志窗口

## 6. 日志与统计

日志会输出每个文件的处理结果，并在结束时汇总：

- 总候选文件
- 已处理文件
- 已转换
- 已是目标编码
- 头注释更新
- 无中文跳过
- 疑似二进制跳过
- 解码失败
- 写入失败
- 扩展名跳过

## 7. 依赖与兼容性

- Python 3.8+（标准库）
- Tkinter
- Windows 环境

## 8. 风险与边界

- 二进制检测为启发式，极少数文本可能被误判
- 只处理文件头连续注释区，不修改正文
- `GB2312` 输出受字符集限制，可能出现写入失败
