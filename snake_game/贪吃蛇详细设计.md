# 贪吃蛇详细设计

## 1. 文档信息

- 项目名称：`snake_game`
- 入口文件：`index.html`
- 文档日期：2026-06-11
- 实现形态：单文件 HTML，内嵌 CSS 与 JavaScript，无外部依赖
- 运行方式：直接打开本地 HTML，或通过本地静态服务访问

## 2. 设计目标

本项目实现一个增强版桌面贪吃蛇小游戏，核心目标如下：

- 保持经典贪吃蛇的低学习成本：方向键/WASD 控制，撞墙或撞到自身结束。
- 提升可玩性：加入随机道具、分数倍率、临时穿行、慢速效果。
- 降低初期难度：初始节拍为 `200ms`，速度增长更平缓。
- 提供本地排行：游戏结束后输入用户名，保存前 8 名排行榜。
- 保持部署简单：不引入 npm、构建工具、CDN 或后端服务。

## 3. 文件结构

snake_game/

  贪吃蛇详细设计.md

`index.html` 包含全部运行逻辑。当前没有拆分 CSS、JavaScript 或资源文件，目的是保证文件可直接复制、打开和归档。

## 4. 页面结构

页面主体为左右两栏布局：

- 左侧 `game-shell`：标题、模式标签、Canvas 棋盘、暂停/Game Over 覆盖层。
- 右侧 `side-panel`：分数状态、控制按钮、当前道具效果、道具图例、排行榜。

关键 DOM 节点：

| 节点 | 用途 |
| --- | --- |
| `canvas#board` | 游戏棋盘绘制区域 |
| `#overlay` | 暂停和 Game Over 覆盖层 |
| `#scoreForm` | Game Over 后提交用户名和分数 |
| `#score` | 当前分数 |
| `#bestScore` | 历史最高分 |
| `#level` | 当前速度等级 |
| `#tick` | 当前逻辑节拍，单位 ms |
| `#leaderboard` | 本地排行榜 |

## 5. 核心参数

| 参数 | 当前值 | 说明 |
| --- | ---: | --- |
| `GRID_SIZE` | 24 | 棋盘为 24x24 网格 |
| `INITIAL_TICK_MS` | 200 | 初始移动节拍 |
| `MIN_TICK_MS` | 95 | 普通加速后的最低节拍 |
| `SPEED_STEP_MS` | 5 | 每级速度缩短 5ms |
| `SPEED_FOOD_STEP` | 2 | 每吃 2 个普通食物升 1 级 |
| `SCORE_PER_FOOD` | 10 | 普通食物基础分 |
| `POWERUP_LIFETIME_MS` | 9000 | 道具在棋盘上的存在时间 |
| `LEADERBOARD_LIMIT` | 8 | 排行榜保留前 8 名 |

慢速时钟会在基础节拍上额外增加 `45ms`，因此它不会改变速度等级，只改变实际移动节拍。

## 6. 状态模型

主要运行状态由闭包内变量维护：

| 变量 | 说明 |
| --- | --- |
| `snake` | 蛇身坐标数组，头部为 `snake[0]` |
| `direction` | 当前生效方向 |
| `queuedDirection` | 下一个 tick 要切换的方向 |
| `food` | 普通食物坐标 |
| `powerup` | 当前棋盘上的随机道具，最多一个 |
| `score` | 当前分数 |
| `eaten` | 已吃普通食物数量，用于速度等级 |
| `growthPending` | 后续还需要增长的蛇身节数 |
| `paused` | 暂停状态 |
| `gameOver` | 游戏结束状态 |
| `scoreSaved` | 当前局分数是否已提交排行榜 |
| `effects` | 当前激活中的道具效果截止时间 |
| `leaderboard` | 本地排行榜缓存 |

方向控制采用 `queuedDirection`，避免同一个 tick 内出现不稳定方向切换；直接反向移动会被拒绝。

## 7. 游戏循环

游戏循环由 `requestAnimationFrame(frame)` 驱动。`frame(now)` 负责累计浏览器帧间隔，并在累计时间达到 `tickMs()` 后执行一个或多个逻辑 tick。

基本流程：

```text
requestAnimationFrame
  -> 计算 delta
  -> 如果未暂停且未结束，累加 accumulator
  -> accumulator >= tickMs() 时执行 step()
  -> draw() 重绘棋盘
  -> 下一帧 requestAnimationFrame
```

使用这种结构可以让渲染频率跟随浏览器刷新率，同时保持蛇的逻辑移动速度稳定。

## 8. 移动与碰撞

每次 `step()` 执行以下逻辑：

1. 清理过期道具。
2. 将 `queuedDirection` 应用到 `direction`。
3. 计算下一格蛇头坐标。
4. 判断是否撞墙。
5. 判断是否吃普通食物或道具。
6. 判断是否撞到自身。
7. 插入新蛇头，按增长状态决定是否移除蛇尾。
8. 更新分数、道具效果、HUD。

普通状态下：

- 撞墙：Game Over。
- 撞自身：Game Over。

相位穿行激活时：

- 越过边界会从对侧出现。
- 自身碰撞检测暂时关闭。

## 9. 食物与道具

普通食物使用 `placeItem()` 在非蛇身位置随机生成。

随机道具生成规则：

- 棋盘上最多存在一个道具。
- 每吃 3 个普通食物尝试生成一个随机道具。
- 道具不与蛇身或普通食物重叠。
- 道具超过 `POWERUP_LIFETIME_MS` 后自动消失。

当前道具定义：

| 类型 | 画面标记 | 效果 |
| --- | --- | --- |
| 黄金食物 | `G` | +30 分，蛇身额外增长 2 节 |
| 慢速时钟 | `S` | 8 秒内移动节拍增加 45ms |
| 双倍分数 | `x2` | 10 秒内所有加分翻倍 |
| 相位穿行 | `P` | 7 秒内穿墙、穿身 |

道具定义集中在 `POWERUPS` 表中，包含显示名称、标记、主色、暗色和高光色，方便后续扩展。

## 10. 计分与排行

计分规则：

- 普通食物：基础 +10 分。
- 双倍分数激活时，普通食物为 +20 分。
- 黄金食物：基础 +30 分，双倍分数激活时为 +60 分。

最高分和排行榜都保存在浏览器 `localStorage`：

| Key | 内容 |
| --- | --- |
| `snake_game_best_score_v1` | 历史最高分 |
| `snake_game_leaderboard_v1` | 排行榜数组 |
| `snake_game_player_name_v1` | 上次输入的用户名 |

排行榜提交流程：

1. 游戏结束后显示 `scoreForm`。
2. 用户输入用户名。
3. 提交后调用 `saveScore()`。
4. 用户名会被裁剪到 16 个字符。
5. 空用户名保存为“匿名玩家”。
6. 排行榜按分数降序排序，只保留前 8 名。

## 11. 渲染设计

Canvas 渲染函数分层如下：

- `drawGrid()`：绘制棋盘底色和 24x24 网格。
- `drawRoundCell()`：绘制蛇身圆角方块。
- `drawSphere()`：绘制带径向渐变、高光、阴影和文字标记的立体道具球。
- `drawFood()`：基于 `drawSphere()` 绘制普通食物。
- `draw()`：组合绘制棋盘、食物、道具和蛇身。

Canvas 会根据设备像素比 DPR 调整实际位图尺寸，避免高分屏下发虚。

## 12. 输入事件

键盘输入：

| 按键 | 功能 |
| --- | --- |
| 方向键 | 控制蛇移动 |
| WASD | 控制蛇移动 |
| 空格 | 暂停/继续 |
| Enter | 重新开始 |

当焦点在用户名输入框内时，全局键盘控制会跳过，避免输入用户名时触发重开。

按钮输入：

- `暂停/继续`：切换暂停状态。
- `重新开始`：重置游戏。
- `保存排名`：提交当前局分数。

## 13. 暂停与可见性

页面监听 `visibilitychange`：

- 当 `document.hidden` 为真时自动暂停。
- 返回页面后不会自动继续，需要玩家手动点击继续或按空格。

该设计避免玩家切换窗口后蛇继续移动导致误死。

## 14. 兼容性与限制

- 依赖现代浏览器 Canvas 2D API。
- 排行榜仅保存在当前浏览器本地，不跨浏览器、不跨设备同步。
- 清除浏览器站点数据会删除最高分和排行榜。
- 当前没有移动端触控控制，主要面向桌面键盘试玩。
- 当前没有音效、关卡、障碍物或联网排行。
